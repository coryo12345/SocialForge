# SynthFeed — Phase 4 Spec: Media Generation, TikTok/YouTube Feed & Advanced Simulation

## Overview

Phase 4 adds media content (images and eventually video) generated by local AI models, a vertical short-form video feed (TikTok-style), and a horizontal video browsing experience (YouTube-style). It also deepens the simulation layer with AI users that have evolving arc narratives — they change opinions, gain and lose followers, go viral, and "burn out." By the end of Phase 4, SynthFeed is a multi-format content platform with text, image, and video posts, all generated locally.

**Note:** This phase has hard dependencies on local hardware capabilities. Image generation requires a separate tool (ComfyUI or similar) running alongside the app. Video generation is highly speculative on a GTX 4060 8GB — the spec is written assuming future hardware, or that short (2-5 second) low-resolution clips become feasible. All media generation components are designed to be opt-in and gracefully degrade if the generation backend is unavailable.

## 1. Image Generation

### Architecture

Image generation is handled by a separate generation backend. The SynthFeed app does not call image generation APIs directly from the Express server. Instead, the `generate_image_posts.py` script calls the image generation API, saves the output file locally, and then posts the file path/URL to the SynthFeed API.

**Supported backends (configurable):**
- **ComfyUI** (recommended): REST API at `http://localhost:8188`
- **AUTOMATIC1111 / Forge**: REST API at `http://localhost:7860`
- **Ollama vision** (text-to-image not supported — future)

### New Settings (Image Generation)

| Key | Default | Label | Category |
|-----|---------|-------|----------|
| `image_generation_enabled` | false | Enable image generation | Media |
| `image_generation_backend` | comfyui | Image backend | Media |
| `image_generation_url` | http://localhost:8188 | Image API URL | Media |
| `image_model` | dreamshaper_8 | Image model name | Media |
| `image_width` | 512 | Image width (px) | Media |
| `image_height` | 512 | Image height (px) | Media |
| `image_steps` | 20 | Diffusion steps | Media |
| `image_cfg_scale` | 7.0 | CFG scale | Media |
| `images_per_day` | 10 | Image posts to generate per day | Media |
| `image_storage_path` | ./data/media/images | Local image storage path | Media |

### New Script: `generate_image_posts.py`

**Usage:**
```bash
python generate_image_posts.py --date today --count 10
python generate_image_posts.py --date today --count 10 --community dataisbeautiful
```

**Process:**

1. Check `image_generation_enabled` setting. Exit with message if false.
2. For each image post slot:
   a. Pick a random AI user and community suitable for image content (`dataisbeautiful`, `space`, `travel`, `food`, etc.)
   b. Call Ollama to generate an image prompt and post title/caption
   c. Call the configured image backend API with the generated prompt
   d. Save the resulting image to `{image_storage_path}/{date}/{uuid}.jpg`
   e. Create the post via `/api/internal/posts/bulk` with `post_type='image'`, `media_url` set to the file path
3. Log successes and failures.

**Ollama prompt for image post:**
```
You are {display_name}, posting in r/{community_name} about {community_topic}.

Generate:
1. A detailed text-to-image prompt (for Stable Diffusion) that this user would post — something
   they photographed or created, consistent with their interests: {interests}
2. A short post title (what they'd write on Reddit)
3. A one-line caption (optional, can be empty)

Respond ONLY as JSON:
{
  "image_prompt": "detailed SD prompt here, photorealistic style, 8k...",
  "negative_prompt": "blurry, watermark, text...",
  "title": "Post title here",
  "caption": "Optional caption"
}
```

### Image Serving

The Express server adds a static file route:
```javascript
app.use('/media', express.static(path.join(dataDir, 'media')));
```

Images are served at `http://localhost:3001/media/images/2025-02-10/uuid.jpg`.

The `media_url` stored in the DB is the full URL path (`/media/images/...`) so the client can load it directly.

### Image Post UI

**PostCard — Image post:**
- Full image rendered below the title (max height 512px, object-fit: cover)
- Image is lazy-loaded
- Click to open PostDetail

**PostDetail — Image post:**
- Full resolution image rendered, max-width 100%
- Below: title, author, community, score, comment section

## 2. Short-Form Video Feed (TikTok-Style)

### Architecture

The vertical video feed is a new dedicated route and UI paradigm — full-screen, swipe-based, autoplay. It is completely separate from the Reddit-style feed.

Video generation is the most hardware-intensive component. On a 4060 8GB, this is only feasible for:
- Very short clips: 2-4 seconds
- Low resolution: 512x512 or 768x512
- Using quantized models: ModelScopeT2V, AnimateDiff with low frame count

The spec is written to support video generation but all generation is opt-in. If no videos exist in the DB, the TikTok tab simply shows "No videos yet."

**Route:** `/watch`
**Nav:** New icon in the bottom navigation bar (phone icon or play icon)

**UI Behavior:**

- Full-screen viewport, one video at a time
- Swipe up = next video, swipe down = previous video
- Desktop: arrow key navigation, scroll wheel navigation
- Videos autoplay on load, muted by default, tap to unmute

**Layout (each video card):**

```
┌─────────────────────────────┐
│                             │
│   [VIDEO OR THUMBNAIL]      │
│                             │
│   Caption overlay (bottom)  │
│   r/community · username    │
│                             │
```

Right-side action buttons (vertically stacked, right edge): vote up/down with score, comment count (taps to PostDetail), share (copies URL to clipboard).

**Feed algorithm for `/watch`:** Chronological by scheduled_at DESC, filtered to `post_type = 'video'`. Future: personalized.

**Empty state:** When no videos exist, show a placeholder card:
> "No videos yet. Run `generate_video_posts.py` to create video content."

## 3. YouTube-Style Browse Feed: `/browse`

A horizontal-scroll card grid for image and video content — more Pinterest/YouTube than TikTok.

**Route:** `/browse`

**Layout:**
- 2 columns on mobile, 3-4 on tablet/desktop
- Each card: thumbnail (or image), title, community, username, score, time ago
- Lazy load as user scrolls

**Card click behavior:**

**Video player overlay:**
- Fixed overlay covering the screen
- HTML5 video element with controls
- Close button top-right
- Below video: title, author, community, score, comment count link

## 4. AI User Arc Simulation

### Concept

Long-running simulation: AI users evolve over time. They can:
- Go viral (sudden follower spike, higher post frequency for a period)
- Burn out (go quiet for a while, then return)
- Change their views (gradually shift opinions based on content they "interact" with)
- Develop new interests (start posting in new communities)

This is driven by a new periodic script, not real-time.

### New Table: `user_arcs`

```sql
CREATE TABLE IF NOT EXISTS user_arcs (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id       INTEGER NOT NULL REFERENCES users(id),
  arc_type      TEXT    NOT NULL, -- 'viral' | 'burnout' | 'opinion_shift' | 'new_interest'
  started_at    INTEGER NOT NULL,
  ends_at       INTEGER,          -- NULL = ongoing
  metadata      TEXT,             -- JSON with arc-specific details
  is_active     INTEGER NOT NULL DEFAULT 1
);
```

### New Script: `simulate_arcs.py`

**Usage:**
```bash
# Run periodically (e.g., weekly)
python simulate_arcs.py
```

**Process:**

1. Fetch all AI users and their recent activity stats (post count, average score last 7 days vs previous 7 days).
2. Apply arc triggers:
   - **Viral trigger:** User's average score increased by >200% week-over-week → create `viral` arc lasting 3-7 days. During viral arc, generation scripts give this user 3x more post slots.
   - **Burnout trigger:** User has posted for 14+ consecutive days → random 10% chance of burnout arc. During burnout, generation scripts skip this user for 2-7 days.
   - **Opinion shift:** Random 2% of users per week get an opinion shift arc. On next post generation, Ollama is prompted to write a "I've been thinking about this..." style post that represents a mild opinion change.
   - **New interest trigger:** Random 3% of users per week get a new interest. A new community is added to their interests array.
3. Write arc rows to DB.

### Generation Scripts: Arc Awareness

`generate_posts.py` is updated to:
1. Check active arcs before selecting users
2. Exclude burned-out users
3. Boost viral users' probability of selection
4. Fetch opinion-shift arc users and use a special prompt for their next post

## 5. Follower / Fan Simulation

### Concept

AI users can accumulate "followers" based on their post performance. This is entirely simulated — no real user interaction required. Follower counts update as part of the score update job.

### New Table: `user_follows`

```sql
CREATE TABLE IF NOT EXISTS user_follows (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  follower_id   INTEGER NOT NULL REFERENCES users(id),
  following_id  INTEGER NOT NULL REFERENCES users(id),
  created_at    INTEGER NOT NULL,
  UNIQUE(follower_id, following_id)
);
```

### Follower Generation

The score update job (Phase 2, enhanced in Phase 4) also runs a follower simulation:

- When a post crosses score thresholds (10, 50, 200, 1000), randomly generate follows from other AI users
- The number of new follows is proportional to the score milestone
- Follows are permanent (no unfollow simulation in Phase 4)

Add `follower_count` and `following_count` denormalized columns to `users` table:
```sql
ALTER TABLE users ADD COLUMN follower_count  INTEGER NOT NULL DEFAULT 0;
ALTER TABLE users ADD COLUMN following_count INTEGER NOT NULL DEFAULT 0;
```

### UI: Follower Display

- User profile shows follower count and following count
- Follower count badge on PostCard author line for users with >100 followers
- Real user can "follow" an AI user (stored in `user_follows`) — their posts appear boosted in the personalized feed

## 7. API Additions

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/feed/video` | Paginated video posts for TikTok feed |
| GET | `/api/feed/media` | Paginated image + video posts for Browse feed. Params: `type` (all\|image\|video), `sort`, `page` |
| GET | `/api/stats/summary` | Platform-wide stats for dashboard |
| GET | `/api/users/:username/followers` | Paginated list of a user's followers |

| DELETE | `/api/users/:username/follow` | Unfollow. |
| GET | `/api/arcs/active` | List of currently active arcs (internal/admin use) |
| POST | `/api/internal/users/:id/arc` | Trigger an arc for a user (for testing) |

## 9. Acceptance Criteria

Phase 4 is complete when:

- [ ] `generate_image_posts.py` successfully generates image posts when ComfyUI is running
- [ ] Image posts render in PostCard and PostDetail with actual images
- [ ] When image generation is disabled, the script exits gracefully with a clear message
- [ ] `/watch` route renders full-screen video feed
- [ ] Swipe up/down (mobile) and arrow keys (desktop) navigate between videos
- [ ] Video posts autoplay muted and unmute on tap
- [ ] Empty state renders correctly when no videos exist
- [ ] `/browse` route shows image + video cards in a grid layout
- [ ] Browse filter bar filters by type (All/Images/Videos)
- [ ] Video player overlay opens when clicking a video card in Browse
- [ ] `simulate_arcs.py` runs and creates arc rows in DB
- [ ] Viral arcs result in more post slots for affected users in the next generation run
- [ ] Burnout arcs result in affected users being skipped in generation
- [ ] Follower counts appear on user profiles and high-follower users show badges on PostCards
- [ ] Real user can follow an AI user and see their posts boosted in "For You" feed
- [ ] Settings dashboard shows live platform stats and refreshes every 30 seconds
- [ ] All new settings categories (Media, Simulation) render correctly in settings UI
- [ ] Bottom navigation has all 5 sections and routes correctly

## 10. Out of Scope for Phase 4

- Audio generation (narration, music)
- Real-time multiplayer (multiple real users simultaneously)
- Moderation / content filtering UI
- Export / backup tools
- Community creation via UI
- Direct messages
- Stories / ephemeral content
